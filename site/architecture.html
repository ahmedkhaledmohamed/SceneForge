    Three decisions that shaped it

    The seams matter more than the boxes.

    seam 1

    One ops layer, two faces

    The CLI and web UI both call ops.py with a log callback. Idempotency, artifact recording, and failure handling exist exactly once — a web button and a terminal command cannot drift apart, because they are the same code.

    seam 2

    The registry is the product surface

    Adding a model is a dict entry: price, capabilities, backend, optional fallback. The CLI's cost estimates, the web UI's dropdowns, and the failover chain all read the same entry — so a new model shows up everywhere at once, priced.

    seam 3

    Files are the only state

    project.json is written atomically after every artifact. A crashed batch loses nothing; a re-run picks up where it stopped; the CLI and the web UI can interleave freely. There is no database to migrate and nothing to host.
