Why it exists

Prompt-by-prompt generation drifts.
Projects shouldn't.

Generate six scenes as six independent prompts and each one finds its own palette, its own light, its own lens. SceneForge captures mood, palette, and lighting once at project creation and composes them into every image and video prompt. Per-scene overrides stay possible; drift doesn't.

The workflow

Profile → outfit → done.

once

Create a profile + character

A profile is your brand — style defaults, a character doll with reference images, and seeds. Characters are global: update the refs once, every project resolves them.

00:00

New project + outfit

One project per post. Add an outfit with clothing item photos and shop links. The project inherits your profile style automatically.

00:01

Process outfit (one click)

Creates pose scenes, generates multi-reference images (character refs + item photos → dressed character), and auto-selects the first option. Or brainstorm scenes with an LLM.

00:05

Refine + select

View images full-size in the lightbox. Refine with prompt editing, model swapping, or regeneration. Cost shown before each operation.

00:06

Generate takes

Batch video clips for all selected scenes at once. Compare takes, mark keepers. 2-6 clips per scene with different motions.

00:14

Export + stitch

Kept takes + links.txt exported as a CapCut-ready folder or zip. Optional: stitch a final cut with crossfades.

Local-first

A project is a folder.

No accounts, no database, no cloud dependency beyond the generation APIs you choose. Profiles and projects are plain directories — profile.json and project.json next to your media. The Studio SPA and CLI read the same files.

~/SceneForge/
└── my-brand/
    ├── profile.json           ← style defaults, character refs, seeds
    ├── refs/characters/pchar-1/ ← shared character doll photos
    └── projects/spring-look/
        ├── project.json       ← scenes, outfits, full prompt history
        ├── images/scene-01/   ← image options per scene
        ├── clips/scene-01/    ← video takes
        └── export/            ← kept takes + links.txt for CapCut
