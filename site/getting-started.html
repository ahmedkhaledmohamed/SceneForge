Getting started

First video in
ten minutes.

Everything runs on your machine; the only cloud involved is the generation API you pick. You'll need Python 3.12+, ffmpeg, and a Together AI key (~$0.05 covers a full test project's images; video clips are priced per model). Prefer to spend nothing while exploring? The built-in fake models run the whole pipeline for $0.

1Install

Clone, create a venv, install. ffmpeg comes from your package manager.

$ git clone https://github.com/ahmedkhaledmohamed/SceneForge.git && cd SceneForge
$ python3 -m venv .venv && source .venv/bin/activate
$ pip install -e .
$ brew install ffmpeg   # or apt install ffmpeg

2Add your key

$ cp .env.example .env
# open .env and set:
TOGETHER_API_KEY=your-key-here

3Create a project

Four questions — concept, mood, palette, lighting — become the project's style anchor, the phrase prepended to every generation from here on.

$ sceneforge create "autumn morning"
Concept (what is this video about?): a cozy autumn morning routine, slow and comforting
Mood: calm, intimate, unhurried
Color palette: muted earth tones, amber, cream
Lighting: warm golden hour light through windows
Created project at ./autumn-morning
$ cd autumn-morning

4Break it into scenes

The LLM proposes; you approve before anything is saved. Or skip the LLM entirely and pass your own with repeated --scene flags.

$ sceneforge add-scenes --count 3
  1. Soft golden light streams through windows, illuminating a warm breakfast table
  2. A person slowly pours steaming coffee into a cream-colored mug
  3. A book lies open on a plush couch, surrounded by autumn leaves and blankets
Add these scenes? [y/N]: y

5Generate and pick images

A few options per scene, each carrying the anchor. Look at the files — or run sceneforge ui and compare them side by side in the browser — then select.

$ sceneforge generate-images --options 2
Generating 6 images with flux-schnell (~$0.02)
$ sceneforge select scene-01 2
$ sceneforge select scene-02 1
$ sceneforge select scene-03 1

6Animate the stills

Image-to-video, per scene. This is the spend step — check sceneforge models for prices first. Re-runs skip finished scenes, so a failed batch just resumes.

$ sceneforge generate-clips
Generating 3 clips with seedance-2.0 (~$2.40)
scene-01: done (5.1s)
scene-02: done (5.1s)
scene-03: done (5.0s)

7Stitch the final cut

$ sceneforge stitch
Final video: output/final.mp4 (7.0s, 720x1280)

2× speed and 0.3s crossfades by default; both are flags. The output is silent by design — music and text belong to your editor.

8Or drive it from the browser

The web UI shares state with the CLI — same project.json, same operations. Selection with thumbnails beats selection with filenames.

$ sceneforge ui --dir ~/videos
SceneForge UI on http://127.0.0.1:8000

9Try everything for free

The fake models are real registry entries backed by ffmpeg — the entire pipeline runs, files and all, for nothing. Good for learning the flow before spending.

$ sceneforge generate-images --model fake-image
$ sceneforge generate-clips --model fake-video
$ sceneforge stitch

10Optional: your own GPU backend

The runpod-wan-i2v and runpod-flux models run on a serverless RTX 4090 you rent per second — roughly $0.12 a clip instead of $0.80, with automatic fallback to the hosted APIs when your endpoint is down.

Setting it up means creating a RunPod account, a network volume for model weights, and a serverless endpoint from the provided Docker worker — the full runbook lives at runpod-worker/README.md. Budget ~$10 of credit and ~$10.50/month for weight storage.
