Getting started

First video in
ten minutes.

Everything runs on your machine; the only cloud is the generation API. You'll need Python 3.12+, ffmpeg, and a Together AI key (~$0.05 covers a full test project). Prefer to spend nothing? The fake models run the whole pipeline for $0.

1Install

Clone, create a venv, install. ffmpeg comes from your package manager.

$ git clone https://github.com/ahmedkhaledmohamed/SceneForge.git && cd SceneForge
$ python3 -m venv .venv && source .venv/bin/activate
$ pip install -e .
$ brew install ffmpeg   # or apt install ffmpeg

2Add your key

$ cp .env.example .env
# open .env and set:
TOGETHER_API_KEY=your-key-here

3Launch Studio

The daily-driver interface — a React SPA over a FastAPI JSON API. Profile-scoped: create a brand, set a style, add your character doll, then create projects under it.

$ sceneforge studio
SceneForge Studio on http://127.0.0.1:8000 (home: ~/SceneForge)

4How reference images work

Each scene owns its reference images. Profile characters are shared across all scenes.

Profile character (your doll)        → sent to every scene using that character
       ↓
Scene refs (garment photos, style)    → owned by the scene, dropped onto the card
       ↓
Project refs (shared background)      → sent to ALL scenes (optional)
       ↓
AI generation = character + scene refs + project refs + scene text → image

Drop reference images directly onto a scene card — garment photos, style examples, background references. Add shop links on each ref for export.

5Create a profile + character

A profile is your brand — style defaults and a character with reference photos. The character is global: update the refs once, every project sees them.

6New project + scenes

One project per post. Add scenes — each one describes a visual moment. Drop reference images (garment photos, style refs) directly onto the scene card. Add shop links on any ref for export.

7Generate images

Click "Generate images" — multi-reference composition uses character refs + scene refs to produce the image. Multiple options per scene, cost shown before you commit.

8Refine + select

Compare images side by side (press 1/2/3 to select). Refine with prompt editing or model swapping. Each regeneration creates a new lane — you see the full history.

9Create clips

Clips are separate from scenes. Click "+ clip" → pick a start image from any scene, optionally an end image, set duration (3-10s), model, and motion prompt → generate. Download completed clips directly for CapCut.

9Or use the CLI

Everything the Studio does is also a CLI command. Good for scripting and batch work.

$ sceneforge create "autumn morning"
$ sceneforge add-scenes --count 6
$ sceneforge generate-images
$ sceneforge select scene-01 2
$ sceneforge generate-clips
$ sceneforge stitch

10Try everything for free

The fake models are real registry entries backed by ffmpeg. The entire pipeline runs — files and all — for $0. Set them in project settings or via CLI:

$ sceneforge generate-images --model fake-image
$ sceneforge generate-clips --model fake-video

11Optional: self-hosted GPU

runpod-wan-i2v runs Wan2.2-TI2V-5B on a rented RTX 4090 — ~$0.10/clip vs $0.80 hosted, with automatic fallback when the endpoint is down.

Full setup: RunPod account, network volume, serverless endpoint from the Docker worker. See runpod-worker/README.md. Budget ~$10 credit + $10.50/mo storage.
